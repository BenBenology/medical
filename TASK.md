## V1: 多轮追问与最小 PDF RAG

### T1.1 Agent 路由判断

- 根据用户输入判断 `answer`、`follow_up`、`escalate`
- 先使用简单规则或轻量 prompt，不追求复杂编排
- 为每种模式定义稳定输出

### T1.2 会话上下文管理

- 保存最小多轮上下文
- 支持将上一轮回答与追问串联
- 控制上下文长度，避免 prompt 失控

### T1.3 PDF 导入与解析

- 约定本地 PDF 存放目录
- 实现最小 PDF 文本提取
- 为解析失败文档提供日志和跳过机制

### T1.4 文本切片与索引

- 设计适合 PDF 指南的切片方式
- 建立最小向量索引或可替换检索接口
- 保留文档名、页码、片段摘要等元信息

### T1.5 检索增强生成

- 在需要时执行知识库检索
- 将检索结果注入回答生成流程
- 控制引用数量，避免噪声过大

### T1.6 证据展示数据结构

- 返回引用编号、标题、片段、来源路径、页码
- 保证前端可直接渲染

## V2: 稳定性与可解释性增强

### T2.1 高风险场景策略

- 定义高风险关键词或症状规则
- 输出升级提示与更明确免责声明
- 避免对高风险场景给出过强结论

### T2.2 错误处理与降级

- LLM 调用失败时返回保守说明
- RAG 失败时允许无引用回答
- 对外保持稳定错误结构

### T2.3 最小 trace 与日志

- 为每次请求生成 trace id
- 记录模式判断、检索开关、错误摘要
- 避免记录敏感原文或完整隐私信息

### T2.4 基础状态持久化

- 评估是否需要轻量会话存储
- 如需要，先使用本地文件或 SQLite 级别方案
