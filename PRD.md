# PRD.md

## 项目名称

Medical Consultation System

## 一句话定位

一个基于 LLM + Agent + RAG 的智能问答或问诊演示系统，帮助用户提问后获得建议和依据。

## 背景

近期很多网络上问诊的软件或者浏览器答案层出不穷，真假难辨。有症状者有相应的怀疑，但是因某些原因无法立马就医，需要一个相对权威的医生或者AI帮忙。

## 目标用户

第一阶段用户：

1. 项目作者自己
2. 面试官或评审老师
3. 想体验 AI 应用 Demo 的同学或朋友

## 核心问题

用户希望：

1. 用户用自然语言提出问题
2. 用户获得相对可信、结构清晰的回答
3. 在必要时看到来源或证据
4. 在高风险场景下得到更谨慎的提示

## 产品目标

MVP 阶段需要做到：

1. 支持基础对话输入输出
2. 支持简单多轮追问或流程控制
3. 支持最小知识库检索
4. 支持展示引用证据
5. 支持前端可视化演示

## 非目标

当前版本不追求：

1. 医疗级准确率承诺
2. 商业化后台系统
3. 多租户与权限体系
4. 大规模生产部署
5. 复杂运营功能

## 核心功能

### 1. 对话入口

用户输入自然语言问题，系统返回回答。

### 2. Agent 决策

系统根据上下文判断当前应：

1. 继续追问
2. 直接回答
3. 给出升级提示

### 3. RAG 检索

系统可基于本地知识库检索相关内容，并将结果作为依据返回。

### 4. 证据展示

前端可展示引用编号、摘要和来源，提升结果可解释性。

### 5. 基础追踪

保留最小 trace、日志或调试信息，便于开发与演示。

## 用户流程

1. 用户打开页面
2. 输入问题
3. 后端接收请求
4. Agent 判断当前模式
5. 如需检索，则执行 RAG
6. 返回最终回答和证据
7. 前端展示结果

## MVP 验收标准

满足以下条件即可视为第一版完成：

1. 本地可启动
2. 前端可发起请求
3. 后端可返回结构化响应
4. 至少有一个稳定主流程
5. 至少有一个最小测试或自测脚本
6. README 能指导别人跑起来

## 版本规划

### V0

1. 跑通单轮问答
2. 完成前后端联调

### V1

1. 加入多轮追问
2. 加入最小知识库检索

### V2

1. 加入证据引用
2. 加入状态持久化
3. 加入更稳定的错误处理

### V3

1. 加入评测脚本
2. 加入更完善的可观测性
3. 优化 UI 和交互

## 风险

1. LLM 输出不稳定
2. Prompt 变复杂后难以维护
3. RAG 数据质量决定上限
4. 一开始做太全会导致项目失控

## 设计原则

1. 先做最小闭环
2. 每个版本只解决少数关键问题
3. 尽量让输出结构化
4. 所有关键能力都保留 fallback
